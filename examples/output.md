# 输出示例

写入成功后，优先使用代码块表格回复。不要使用 Markdown 表格，因为普通消息场景里容易错位。

## 单笔记录

```text
记账成功！

       名称            金额     类型    账本    项目归属     标签
─────────────────────────────────────────────────────────────────
     东方树叶          5.00     支出    个人       无        食物

账本链接：{{ledger_url}}
```

## 多笔记录

```text
记账成功！

       名称            金额     类型    账本    项目归属     标签
─────────────────────────────────────────────────────────────────
     东方树叶          5.00     支出    个人       无        食物
       打车           12.50     支出    个人       无        交通
     微博投流        300.00     支出    工作       无       推广费

账本链接：{{ledger_url}}
```

## 推断示例

输入：

```text
小米微博投流 300
```

推断：

```text
名称：小米微博投流
金额：300.00
类型：支出
账本：工作
日期：记录创建时的北京时间
```

## 失败示例

```text
记账失败：未能读取飞书账本配置。

请先在 SKILL.md 的“账本信息”中补充 APP_TOKEN、TABLE_ID 和外链，或告诉我需要绑定的飞书多维表格链接。
```

## 回滚示例

输入：

```text
退回 CHG-20260429-151552 前版本
```

处理结果：

```text
已根据 backup/CHG-20260429-151552/ 中的变更前备份恢复对应文件，并在 log.md 追加回滚记录。
```
